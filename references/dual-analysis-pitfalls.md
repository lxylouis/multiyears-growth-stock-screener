# 双维度分析：数据合并陷阱

## 核心问题

dual 模式的合并逻辑：

```python
merged = price_df[['代码','名称','行业','分数']]      # 仅含有效股价数据的股票
merged = merged.merge(rev_df[['代码','分数']], how='left')  # 左连接营收
```

**结果：只包含有完整股价数据的股票。** 营收筛选通过但缺少股价历史的股票不会出现在合并表中。

## 实例：沪深300（`window_years=3`, `num_windows=5`, `cond_b_threshold=1.5`）

| 数据集 | 股票数 | 说明 |
|:---|---:|:----|
| 股价有效数据 | 226 | 有完整 `window_years` 年股价的沪深300成分股 |
| 营收有效数据 | 279 | 有年报营收的成分股（含次新股） |
| **股价筛选通过** | **134** | 最新期 > cond_b_threshold ∪ 全期 ≥ cond_a_min |
| **营收筛选通过** | **251** | 同上规则 |
| **双维度合并表** | **226** | 左连接后行数 = 股价有效数据数 |
| 双通过 | 127 | 股价+营收都通过 |
| 仅股价 | 7 | 股价通过但营收未通过 |
| 仅营收 | 76 | 营收通过但股价未通过 |
| 营收通过但不在合并表中 | 48 | = 251 - 127 - 76 = 仅存在于营收数据中的次新股 |

## 哪些股票会缺失？

主要三类：

1. **科创板 (688xxx)** — 2019年7月开板，至今不足7年
2. **创业板次新股** — 2020年后上市的创业板股
3. **主板近5年IPO** — 中国电信(601728)、邮储银行(601658)等

这些股票虽然有完整的年报数据（营收），但上市时间不足 `window_years` 年，无法计算 `window_years` 年股价倍率。

## 对结果解读的影响

| 看到的现象 | 实际原因 |
|:----------|:--------|
| "营收通过236只，双维度营收侧只有182只" | 54只营收通过的次新股缺少股价数据，不在合并表中 |
| "恒指双通过19只，仅营收46只" | 港股很多次新股同样缺少 `window_years` 年股价历史 |
| "某股票营收排名很高但不出现在双维度" | 它可能根本没有进入合并表 |

## 修正措施

在报告中补充表注：**"营收侧通过数含N只缺少股价历史数据的次新股（上市不足 `window_years` 年），实际进入双维度分析的营收通过数为M只"**

或者修改代码在 dual 报告开头加一行说明：

```python
rev_only_missing = len(set(rev_pp['代码']) - set(price_df['代码']))
if rev_only_missing:
    report.add_paragraph(
        f"注：{rev_only_missing}只营收通过股票因缺乏{window_years}年股价历史未纳入双维度分析"
    )
```

## 什么时候这不是问题？

- 当股价数据完整率接近100%（如大部分标的上市超过 `window_years` 年）
- 当只关心"既有股价又有营收"的成熟标的（当前设计意图）
